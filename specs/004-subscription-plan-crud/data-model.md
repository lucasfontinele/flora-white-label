# Data Model: CRUD de Planos de Assinatura Master

## SubscriptionPlan

**Type**: Domain entity, independent entity, not Aggregate Root.

**Purpose**: Represents a platform-level subscription offer available for
organizations to reference.

**Fields**:

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `id` | string | yes | Stable entity identifier generated by the domain/base entity or persistence. |
| `title` | string | yes | Trim before validation and persistence; must not be blank. |
| `description` | string or null | no | May be omitted or null; if supplied as string, trim and reject when blank after trim. |
| `price` | `MoneyInCents` | yes | Domain value object wrapping integer cents. |
| `operatorsLimit` | number | yes | Integer greater than zero. |
| `patientsLimit` | number | yes | Integer greater than zero. |

**Persistence fields**:

| Field | Type | Notes |
|-------|------|-------|
| `priceInCents` | integer | Prisma/database representation of `price.value`. |
| `createdAt` | DateTime | Persistence-generated timestamp returned in API responses. |
| `updatedAt` | DateTime | Persistence-generated timestamp returned in API responses. |

**Relationships**:

- `Organization.currentPlanId` references `SubscriptionPlan.id` through the
  existing Prisma relationship.
- `SubscriptionPlan` does not own organizations and does not receive
  `organizationId`.

**Validation**:

- Title cannot be missing or blank after trim.
- Description can be absent/null; blank string is invalid.
- Price must be an integer number of cents and cannot be negative.
- Operator and patient limits must be positive integers.

**State transitions**:

- Created: plan is persisted and becomes available in the master catalog.
- Updated: editable fields are replaced as a full update.
- Deleted: plan is removed only when no Organization references it.
- Delete blocked: plan remains unchanged when at least one Organization uses it.

## MoneyInCents

**Type**: Domain value object.

**Purpose**: Protects monetary precision by representing money as integer cents.

**Fields**:

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `value` | number | yes | Must be an integer; must be zero or positive. |

**Validation**:

- Reject floats/decimals.
- Reject negative values.
- Accept zero because the current value object allows it.

## SubscriptionPlanReadModel

**Type**: Application DTO/read model.

**Purpose**: Carries API response data without making timestamps domain
invariants.

**Fields**:

| Field | Type | Required | Source |
|-------|------|----------|--------|
| `id` | string | yes | Domain/persistence identifier. |
| `title` | string | yes | Domain entity. |
| `description` | string or null | no | Domain entity/persistence record. |
| `priceInCents` | number | yes | `MoneyInCents.value`. |
| `operatorsLimit` | number | yes | Domain entity. |
| `patientsLimit` | number | yes | Domain entity. |
| `createdAt` | Date | yes | Persistence record. |
| `updatedAt` | Date | yes | Persistence record. |

## Request Models

### CreateSubscriptionPlanRequest

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `title` | string | yes | Nonblank after trim. |
| `description` | string or null | no | Optional/null; nonblank after trim when string. |
| `priceInCents` | number | yes | Integer, zero or positive, no decimals. |
| `operatorsLimit` | number | yes | Positive integer. |
| `patientsLimit` | number | yes | Positive integer. |

### UpdateSubscriptionPlanRequest

Same fields and validation as create. `PUT` is a full replacement, so all
required fields must be present.

### SubscriptionPlanParams

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `id` | string | yes | Nonblank route identifier; use UUID validation if it matches project route conventions during implementation. |

## Repository Port Target

The application repository should preserve current Organization compatibility
and add CRUD support:

```ts
interface SubscriptionPlanRepository {
  findById(id: string): Promise<SubscriptionPlan | null>;
  findDetailsById(id: string): Promise<SubscriptionPlanReadModel | null>;
  findAllDetails(): Promise<SubscriptionPlanReadModel[]>;
  create(plan: SubscriptionPlan): Promise<SubscriptionPlanReadModel>;
  save(plan: SubscriptionPlan): Promise<SubscriptionPlanReadModel>;
  delete(id: string): Promise<void>;
  hasOrganizations(id: string): Promise<boolean>;
}
```

The exact names may be adjusted during implementation, but the responsibilities
should remain the same.
