# Data Model: CRUD de Organizações Master

## Organization

**Type**: Aggregate Root.

**Purpose**: Represents an association/organization tenant registered by the
master backoffice.

**Fields**:

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `id` | string | yes | Stable identifier generated by the domain/base entity or persistence. |
| `tradeName` | string | yes | Trim before validation and persistence; must not be blank. |
| `legalName` | string | yes | Trim before validation and persistence; must not be blank. |
| `cnpj` | `Cnpj` | yes | Normalized to 14 digits; unique across organizations. |
| `primaryCnae` | `Cnae` | yes | Normalized to 7 digits. |
| `secondaryCnaes` | `Cnae[]` | no | Optional input; stored as a list, default empty. |
| `currentPlanId` | string | yes | Must reference an existing `SubscriptionPlan`. |
| `addressId` | string | yes | Must reference the address created or updated with the organization. |

**Persistence fields**:

| Field | Type | Notes |
|-------|------|-------|
| `createdAt` | DateTime | Persistence-generated timestamp returned when available. |
| `updatedAt` | DateTime | Persistence-generated timestamp returned when available. |

**Relationships**:

- Belongs to the current plan through `currentPlanId`.
- Belongs to one persisted address through `addressId`.
- May have settings and future tenant-owned operational records, but this CRUD
  must not expose or manage them.

**Validation**:

- `tradeName` and `legalName` cannot be missing or blank after trim.
- `cnpj` must be valid after normalization and unique.
- `primaryCnae` must be structurally valid.
- `secondaryCnaes` items must each be structurally valid when present.
- `currentPlanId` and `addressId` cannot be blank.

**State transitions**:

- Created: organization and address are persisted in one logical operation.
- Updated: organization and associated address editable fields are replaced in
  one logical operation.
- Deleted/removed: organization disappears from CRUD reads. Its associated
  address must not remain available as an orphan within this resource.
- Rejected: invalid payload, invalid CNPJ/CNAE/CEP/UF, duplicate CNPJ, missing
  organization, or missing plan leaves prior data unchanged.

## Address

**Type**: Persistable Entity, not Aggregate Root.

**Purpose**: Stores the organization address created and maintained through the
organization CRUD.

**Fields**:

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `id` | string | yes | Stable entity identifier. |
| `title` | string or null | no | Optional; trim when provided; no persisted empty string. |
| `zipcode` | string | yes | Normalize to digits only; exactly 8 digits. |
| `street` | string | yes | Trim; must not be blank. |
| `neighborhood` | string | yes | Trim; must not be blank. |
| `complement` | string or null | no | Optional; trim when provided; no persisted empty string. |
| `city` | string | yes | Trim; must not be blank. |
| `state` | string | yes | Trim and uppercase; must be a valid Brazilian UF. |

**Relationships**:

- Referenced by `Organization.addressId`.
- No standalone HTTP endpoints in this feature.

## SubscriptionPlan

**Type**: Existing independent Entity.

**Purpose**: Represents the organization's current commercial plan.

**Fields used by this feature**:

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `id` | string | yes | Must exist for create/update. |
| `title` | string | yes | Returned in current plan summary. |
| `priceInCents` | number | yes | Returned as integer cents. |
| `operatorsLimit` | number | yes | Returned in current plan summary. |
| `patientsLimit` | number | yes | Returned in current plan summary. |

**Relationships**:

- `Organization.currentPlanId` references `SubscriptionPlan.id`.
- This feature does not create, update, delete, seed, bill, or charge plans.

## Cnpj

**Type**: Value Object.

**Rules**:

- Accept masked or unmasked input.
- Remove non-numeric characters.
- Store exactly 14 digits.
- Reject blank values, repeated digit sequences, invalid length, and invalid
  check digits.

## Cnae

**Type**: Value Object.

**Rules**:

- Accept formats such as `8630-5/03` and `8630503`.
- Remove non-numeric characters.
- Store exactly 7 digits.
- Validate only structural format; do not consult an official CNAE database.

## OrganizationReadModel

**Type**: Application DTO/read model.

**Purpose**: Carries response data without adding persistence timestamps or
nested response concerns to the aggregate.

| Field | Type | Required | Source |
|-------|------|----------|--------|
| `id` | string | yes | Organization record. |
| `tradeName` | string | yes | Organization aggregate. |
| `legalName` | string | yes | Organization aggregate. |
| `cnpj` | string | yes | `Cnpj.value`. |
| `primaryCnae` | string | yes | `Cnae.value`. |
| `secondaryCnaes` | string[] | yes | `Cnae.value` list. |
| `currentPlan` | CurrentPlanSummary | yes | Subscription plan read data. |
| `address` | AddressSummary | yes | Address read data. |
| `createdAt` | Date | no | Persistence record when available. |
| `updatedAt` | Date | no | Persistence record when available. |

## Request Models

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `organization.tradeName` | string | yes | Nonblank after trim. |
| `organization.legalName` | string | yes | Nonblank after trim. |
| `organization.cnpj` | string | yes | Valid `Cnpj`. |
| `organization.primaryCnae` | string | yes | Valid `Cnae`. |
| `organization.secondaryCnaes` | string[] | no | Defaults to empty list. |
| `organization.currentPlanId` | string | yes | Existing subscription plan. |
| `address.title` | string or null | no | Optional. |
| `address.zipcode` | string | yes | Exactly 8 digits after normalization. |
| `address.street` | string | yes | Nonblank after trim. |
| `address.neighborhood` | string | yes | Nonblank after trim. |
| `address.complement` | string or null | no | Optional. |
| `address.city` | string | yes | Nonblank after trim. |
| `address.state` | string | yes | Valid Brazilian UF. |

### UpdateOrganizationRequest

Same shape and validation as create. `PUT` is a full replacement, so all
required organization and address fields must be present.

### OrganizationParams

| Field | Type | Required | Rules |
|-------|------|----------|-------|
| `id` | string | yes | Nonblank route identifier; use UUID validation only if it matches project route conventions during implementation. |

## Repository Port Target

The organization repository should support current create behavior and the new
CRUD read/write responsibilities:

```ts
interface OrganizationRepository {
  findByCnpj(cnpj: Cnpj): Promise<Organization | null>;
  findByCnpjExcludingId(cnpj: Cnpj, id: string): Promise<Organization | null>;
  findById(id: string): Promise<Organization | null>;
  findDetailsById(id: string): Promise<OrganizationReadModel | null>;
  findAllDetails(): Promise<OrganizationReadModel[]>;
  create(organization: Organization): Promise<void>;
  save(organization: Organization): Promise<void>;
  delete(id: string): Promise<void>;
}
```

The address repository should support the organization use cases:

```ts
interface AddressRepository {
  create(address: Address): Promise<void>;
  save(address: Address): Promise<void>;
  delete(id: string): Promise<void>;
}
```

The exact method names may be adjusted during implementation, but these
responsibilities should remain.
